Upgrade /local/bin/hugo from the github repo.  Get the "extended" version.

This default directory structure is created by:

	hugo new site long-posts && cd long-posts

Complete my setup:

	mkorigf hugo.toml

	cp -rp ../old/.git* ../old/hugo.toml .

	hugo mod init github.com/rebelga/long-posts

	hugo mod tidy

	hugo mod get github.com/theNewDynamic/gohugo-theme-ananke/v2

	ln -s ~/go/pkg/mod/github.com/the\!new\!dynamic/gohugo-theme-ananke/v2\@v2.12.1 ananke

My content is in content/ and in static/images/:

	cp -rpv ../old/content ../old/static .

I have modified the ananke theme with index.html override in layouts/ :

	cp -rpv ../old/layouts .
	
	- add "Recent Posts" <h1> header as in earlier versions

	- add "All Posts" button which is supposed to be there but
	  broken in this ananke version (2.12.1)

	- add twitter:card partial for "summary_large_image" (for Mastodon)
	  also add twitter:image and add og:image

Test with:

	- rm -rf public/ (it will be rebuilt)

	- hugo server -D (automatically rebuilds on any changes)

	- http://localhost:1313 (vs. https://george.rcsocial.net)

When ready to upgrade:

	- rebuild public:  rm -rf public; hugo

	- update git

		> git status	(to see what has changed)
		> git add -A	(to add changes)
		> git commit -m "Upgrade to Hugo v0.157"
		> git push origin main
